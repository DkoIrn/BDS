<task type="auto">
  <name>Task 2: File upload zone with queue management and direct Supabase upload</name>
  <files>src/components/files/file-upload-zone.tsx</files>
  <action>
Create a "use client" component `FileUploadZone` that receives `{ jobId: string, userId: string }` props.

**Drop zone UI (react-dropzone):**
- Use `useDropzone` hook with accept={ACCEPTED_FILE_TYPES}, maxSize={MAX_FILE_SIZE}, multiple=true, noClick=false, noKeyboard=false
- Drop zone visual: dashed border (border-dashed border-2), rounded-lg, centered content with Upload icon (lucide-react), text "Drag and drop files here, or click to browse", and a hint line "CSV, XLS, XLSX - Max 50MB per file"
- States via isDragActive/isDragAccept/isDragReject from useDropzone:
  - Default: border-muted-foreground/25, bg-muted/50
  - isDragActive + isDragAccept: border-teal-500 (brand teal #14B8A6), bg-teal-50, text changes to "Drop files here"
  - isDragReject: border-destructive, bg-destructive/5, text changes to "File type not accepted"
- onDrop callback adds accepted files to queue state (as FileUploadItem with status='queued')
- onDropRejected shows inline errors per rejected file (file-too-large, file-invalid-type) using sonner toast

**Queue state management (useState):**
- State: `uploadItems: FileUploadItem[]` -- array of queued/uploading/completed files
- Each item gets a unique id (crypto.randomUUID())
- Queue display: Below the drop zone, show a list of queued items. Each shows: file icon (FileSpreadsheet), filename, file size (formatted: KB/MB), status indicator, and action button
  - queued: X button to remove from queue
  - uploading: progress bar (indeterminate or percentage if available) + Cancel button
  - uploaded: green Check icon
  - failed: red X icon + error message + Retry button
  - cancelled: "Cancelled" text + Retry button
- "Upload All" button: visible when any items have status='queued'. Styled as primary Button. Positioned above the queue list, right-aligned.
- "Clear All" button: visible when queue has items. Secondary/ghost style. Clears all non-uploading items from queue.

**Duplicate detection:**
- Before adding files to queue, check if any filename already exists in the existing files (passed as prop or fetched) OR in the current queue
- If duplicate found, show a Dialog with options: "Replace" (remove existing, upload new), "Keep Both" (auto-rename by prepending timestamp), "Skip" (don't add to queue)
- For simplicity in MVP: handle one duplicate at a time if multiple duplicates in a batch

**Upload logic (Upload All button handler):**
- Process queued items sequentially (one at a time to avoid overwhelming the connection)
- For each file:
  1. Set status to 'uploading'
  2. Create storage path: `${userId}/${jobId}/${Date.now()}-${file.name}`
  3. Upload to Supabase Storage: `createClient().storage.from('datasets').upload(path, file, { cacheControl: '3600', upsert: false })`
  4. On storage success: call `createFileRecord({ jobId, fileName: file.name, fileSize: file.size, mimeType: file.type, storagePath: path })`
  5. On createFileRecord success: set status to 'uploaded'
  6. On storage or record failure: set status to 'failed', set error message. If DB insert failed but storage succeeded, clean up storage (delete the uploaded file)
- Cancel: Use AbortController per upload. Store controller ref in a Map keyed by item id. On cancel click, call controller.abort(), set status to 'cancelled'
- Retry: Reset status to 'queued', clear error. The next "Upload All" click will pick it up. OR immediately start uploading just that file.

**After successful uploads:** Keep uploaded items visible with checkmarks. The drop zone remains above for adding more files. Call `router.refresh()` to re-fetch the file list from the server.

**Progress:** Supabase standard upload may not expose granular progress. Use an indeterminate progress bar (animated stripe) during upload. If the SDK's upload returns a promise without progress events, that is acceptable for MVP.
