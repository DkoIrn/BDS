       - Props: { run: ValidationRun, onRerun: () => void }
       - Renders a card with:
         - Header: "Validation Results" with run timestamp
         - 3 severity count badges in a row: Critical (red), Warning (yellow), Info (blue)

    3. Update src/components/files/file-detail-view.tsx:
       - Add state: validating (boolean), validationRun (ValidationRun | null), validationError (string | null)
       - Add state: datasetStatus (DatasetStatus) initialized from dataset.status -- track locally for optimistic updates
       - After the "Confirm Mappings" / "Edit Mappings" button area, when confirmed === true AND datasetStatus is "mapped":
         - Show "Run QC" button (Play icon from lucide-react)
         - Clicking triggers: set validating=true, set datasetStatus="validating", fetch POST /api/validate with { datasetId: dataset.id }
         - On success: parse response as ValidateResponse, set validationRun from response data, set datasetStatus="validated", set validating=false, toast.success("Validation complete")
         - On error: set validationError with message, set datasetStatus="validation_error", set validating=false, toast.error(message)
       - When validating === true: show ValidationProgress component with "Running validation..."
       - When datasetStatus === "validated" AND validationRun exists: show ValidationSummary
       - When datasetStatus === "validation_error": show error banner (similar to parse error style)
       - On component mount: if dataset.status is "validated", fetch latest validation run via getValidationRuns(dataset.id) and display summary
       - handleRerun function: same as Run QC logic (allows re-running from summary)
       - The "Run QC" button should ONLY appear when mappings are confirmed (confirmed === true)
