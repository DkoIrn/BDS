       validation_runs table:
       - id UUID PK default gen_random_uuid()
       - dataset_id UUID REFERENCES datasets(id) ON DELETE CASCADE NOT NULL
       - run_at TIMESTAMPTZ DEFAULT NOW()
       - total_issues INTEGER NOT NULL DEFAULT 0
       - critical_count INTEGER NOT NULL DEFAULT 0
       - warning_count INTEGER NOT NULL DEFAULT 0
       - info_count INTEGER NOT NULL DEFAULT 0
       - pass_rate REAL
       - completeness_score REAL
       - status TEXT NOT NULL DEFAULT 'running'
       - created_at TIMESTAMPTZ DEFAULT NOW()
       - Index on dataset_id
       - RLS enabled with SELECT policy: users can view runs for their own datasets (via datasets.user_id = auth.uid())

       validation_issues table:
       - id UUID PK default gen_random_uuid()
       - run_id UUID REFERENCES validation_runs(id) ON DELETE CASCADE NOT NULL
       - dataset_id UUID REFERENCES datasets(id) ON DELETE CASCADE NOT NULL
       - row_number INTEGER NOT NULL
       - column_name TEXT NOT NULL
       - rule_type TEXT NOT NULL
       - severity TEXT NOT NULL
       - message TEXT NOT NULL
       - expected TEXT
       - actual TEXT
       - kp_value REAL
       - created_at TIMESTAMPTZ DEFAULT NOW()
       - Indexes on run_id, dataset_id, severity
       - RLS enabled with SELECT policy: users can view issues for their own datasets

    1. Create src/app/api/validate/route.ts following the existing parse route pattern:
       - POST handler with auth check (createClient, getUser, 401 if not authenticated)
       - Extract datasetId from request body
       - Verify dataset ownership: query datasets where id=datasetId AND user_id=user.id, return 404 if not found
       - Verify dataset status is "mapped" (only mapped datasets can be validated), return 400 if not
       - Update dataset status to "validating"
       - Call FastAPI: fetch(`${process.env.FASTAPI_URL}/api/v1/validate`, { method: "POST", headers: {"Content-Type": "application/json"}, body: JSON.stringify({ dataset_id: datasetId }) })
       - If FastAPI returns error: update dataset status to "validation_error", return 500 with error message
       - If FastAPI succeeds: return the FastAPI response JSON (which includes run_id, counts, pass_rate, status)
       - Set maxDuration = 120 (validation may take longer than parsing)
       - IMPORTANT: Do NOT update status to "validated" here -- FastAPI handles that after writing results

    2. Create src/lib/actions/validation.ts with server actions:
       - getValidationRuns(datasetId: string): fetch validation_runs for a dataset, ordered by run_at DESC
       - getValidationIssues(runId: string): fetch validation_issues for a run, ordered by severity (critical first), then row_number
       - Both use createClient(), getUser(), and verify ownership through dataset join

    3. Add FASTAPI_URL to .env.local (or .env.example) with placeholder: FASTAPI_URL=http://localhost:8000
       Note: Do NOT use NEXT_PUBLIC_ prefix -- this is server-side only.
