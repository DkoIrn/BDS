  <action>
    1. Create `backend/app/routers/reports.py` with `router = APIRouter(prefix="/api/v1", tags=["reports"])`.

    2. `GET /api/v1/report/pdf/{run_id}`:
       - Fetch validation_run by run_id from Supabase. 404 if not found.
       - Fetch all validation_issues for that run_id, ordered by row_number.
       - Fetch dataset record (for file_name as dataset_name).
       - Call `generate_pdf_report(run_data, issues, dataset_name)`.
       - Return `StreamingResponse(io.BytesIO(pdf_bytes), media_type="application/pdf", headers={"Content-Disposition": f"attachment; filename=qc-report-{run_id[:8]}.pdf"})`.
       - Use sync def (not async def) -- same pattern as validation router for supabase-py compatibility.

    3. `GET /api/v1/export/dataset/{dataset_id}`:
       - Accept query param `format` (csv or xlsx, default csv) and `run_id` (optional -- uses latest run if not provided).
       - Fetch dataset record. 404 if not found.
       - Download file from Supabase Storage, parse into DataFrame (same pattern as validation.py run_validation_background: CSV via pd.read_csv, Excel via pd.read_excel, using header_row_index and dtype=str).
       - If run_id provided, fetch issues for that run. If not, fetch latest run's issues.
       - For csv: call export_annotated_csv, return StreamingResponse with media_type text/csv.
       - For xlsx: call export_annotated_excel, return StreamingResponse with media_type application/vnd.openxmlformats-officedocument.spreadsheetml.sheet.
       - Content-Disposition filename: `{dataset_filename_stem}-annotated.{format}`

    4. Register router in `backend/app/main.py`: `from app.routers.reports import router as reports_router` and `app.include_router(reports_router)`.
