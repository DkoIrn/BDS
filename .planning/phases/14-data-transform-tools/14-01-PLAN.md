    - merge_datasets([result_a, result_b]) with same columns -> combined rows, same headers
    - merge_datasets([result_a, result_c]) with different columns -> union headers, padded rows

    - split_by_kp with no KP column -> error/warning
    - split_by_column(result, "survey_type") -> one ParseResult per unique value
    - split_by_column with empty column values -> skips empty groups with warning

    1. Create backend/app/routers/transform.py with APIRouter(prefix="/api/v1/transform", tags=["transform"]):

       POST /crs endpoint:
       - Accepts: file (UploadFile), source_epsg (Form, optional), target_epsg (Form, required), target_format (Form, default "csv")
       - Parse file with dispatch_parser. Detect coordinate columns.
       - If source_epsg not provided, run detect_crs_from_values. If detection fails, return 400 with message asking user to select source CRS.
       - Run transform_crs(result, source_epsg, target_epsg).
       - Write output using WRITERS[target_format] (same dict as conversion.py).
       - Return StreamingResponse with Content-Disposition, X-Transform-Warnings, X-Row-Count, X-Source-CRS, X-Target-CRS headers.

       POST /merge endpoint:
       - Accepts: files (List[UploadFile]), target_format (Form, default "csv")
       - Validate at least 2 files. Parse each with dispatch_parser.
       - Run merge_datasets(results).
       - Write output and return StreamingResponse with same header pattern.

       POST /split endpoint:
       - Accepts: file (UploadFile), mode (Form: "kp" or "column"), target_format (Form, default "csv")
       - For "kp" mode: ranges (Form, JSON string of [[start,end],...])
       - For "column" mode: column_name (Form)
       - Parse file. Run appropriate split function.
       - If single output, return StreamingResponse.
       - If multiple outputs, write each to target format, create ZIP using zipfile+BytesIO, return as application/zip with individual file info in X-Split-Files header.
