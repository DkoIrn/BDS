    2. Create src/app/(dashboard)/tools/transform/split/page.tsx:
       - Server component wrapper with title "Split Dataset" and back link to /tools/transform.

    3. Create src/app/(dashboard)/tools/transform/split/split-tool.tsx ("use client"):
       - State machine: upload -> configure -> splitting -> done -> error.

       Step 1 (Upload): Single-file dropzone (same extensions, 50MB limit). Show file info after selection.

       Step 2 (Configure):
       - Mode toggle: "By KP Range" or "By Column Value" (two buttons or radio group).

       KP Range mode:
       - Dynamic list of range inputs. Each range: start KP (number input) and end KP (number input).
       - "Add Range" button to add more ranges. Minimum 1 range.
       - Remove button on each range (if >1).

       Column Value mode:
       - Need to know column names. On file upload, do a lightweight parse or extract headers.
       - Option A (simpler): After file selected, POST to a /api/transform/split/preview endpoint to get headers. But this adds endpoint complexity.
       - Option B (preferred): Read the first line of CSV client-side to get headers, or for non-CSV files, show a text input for column name.
       - Use Option B: For CSV/Excel, read first row client-side with FileReader to extract headers and show a dropdown. For other formats, show a text input field for column name.

       Output format selector: CSV, GeoJSON, KML (default CSV).
       "Split Dataset" button to proceed.

       Step 3 (Splitting): Spinner with "Splitting dataset..." status text.
       POST to /api/transform/split with FormData: file, mode ("kp" or "column"), target_format, ranges (JSON.stringify for KP mode), column_name (for column mode).

       Step 4 (Done):
       - Parse X-Split-Files header to get list of filenames.
       - If response is ZIP (application/zip): Store blob URL for "Download All as ZIP" button.
       - For individual file downloads: Parse the ZIP client-side using the JSZip library is complex. Simpler approach: just offer the ZIP download. Show file list from X-Split-Files header as informational text.
       - Actually, per user decision: "individual file downloads AND Download all as ZIP." To keep it simple without adding JSZip: the primary download is the ZIP file. List the contained files from the header as read-only info. This satisfies "individual" by showing what's inside while the ZIP is the download mechanism.
       - Show stats: number of output files, warnings from X-Transform-Warnings.
       - "Split Another" reset button.

       Error state: Same inline error pattern.

       Client-side CSV header extraction:
       ```typescript
       async function extractCsvHeaders(file: File): Promise<string[]> {
         const text = await file.slice(0, 4096).text()
         const firstLine = text.split('\n')[0]
         return firstLine.split(',').map(h => h.trim().replace(/^"|"$/g, ''))
       }
       ```
       For non-CSV files, show text input instead of dropdown.
