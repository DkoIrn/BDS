**Create `components/data-table-panel.tsx`:**
Floating panel at the bottom of the viewport (absolute, bottom-0, left/right with padding, z-10):
- Hidden by default. Toggle button in the main UI labeled "Data Table" or table icon
- When visible, shows a scrollable table (max-height ~250px) of the active layer's feature attributes
- Table columns: auto-detected from `feature.properties` keys of the first feature in the active layer
- Each row represents one feature
- Click a row: calls `onZoomToFeature(featureIndex: number)` callback
- Header shows layer filename and feature count
- Close/minimize button to collapse back
- If no active layer selected, show "Select a layer to view its data" message
- Style: white background, subtle border-top, overflow-y auto for scrolling

**Create `components/measurement-tool.tsx`:**
This is a sub-component rendered INSIDE the Leaflet map (it uses `useMap` and `useMapEvents`):
- Controlled by `active: boolean` prop and `onToggle` callback
- When active, cursor changes to crosshair (`map.getContainer().style.cursor = "crosshair"`)
- Click on map adds a point to the measurement path (array of LatLng)
- Render a `Polyline` between all clicked points (dashed red line, weight 2)
- Render `CircleMarker` at each clicked point
- Show total distance in a floating label near the last point
  - Use `latlng.distanceTo(prevLatlng)` for each segment, sum for total
  - Format: meters if < 1000, km if >= 1000 (e.g., "1,234 m" or "5.67 km")
- Double-click or pressing Escape finishes/resets the measurement
- A small "Clear" button appears when measurement points exist
- When deactivated (active=false), clear all measurement state and remove polyline/markers
- Restore cursor to default on deactivate

The toggle button for measurement lives in the map-visualizer.tsx UI (floating button near top-right, ruler icon). When toggled on, pass `active={true}` to the measurement tool inside the map.

**Create `components/screenshot-button.tsx`:**
Floating button (absolute positioned, near zoom-to-fit button area):
- Uses `leaflet-simple-map-screenshoter` plugin
- On the LeafletMap component mount, create a screenshoter instance:
  ```typescript
  import { SimpleMapScreenshoter } from "leaflet-simple-map-screenshoter"
  // Inside a useEffect after map is available:
  const screenshoter = new SimpleMapScreenshoter({ hidden: true }).addTo(map)
  ```
  Store the screenshoter ref via `useImperativeHandle` or expose via callback.
- Alternatively, simpler approach: the screenshot button lives INSIDE the leaflet-map.tsx (since it needs `useMap()`). Use `useMap()` to get the map instance, create screenshoter on mount.
- On click: `screenshoter.takeScreen("blob")` returns a promise with a Blob
- Create a blob URL and trigger download with filename `map-screenshot-{timestamp}.png`
- Show brief loading state on button during capture
- If `leaflet-simple-map-screenshoter` has type issues, add a minimal declaration:
  ```typescript
  declare module "leaflet-simple-map-screenshoter" {
    import L from "leaflet"
    export class SimpleMapScreenshoter extends L.Control {
      constructor(options?: { hidden?: boolean })
      addTo(map: L.Map): this
      takeScreen(format?: string): Promise<Blob>
    }
  }
  ```

**Update `map-visualizer.tsx`:**
- Add state: `showDataTable: boolean` (default false), `measurementActive: boolean` (default false)
- Add toggle buttons in the floating toolbar area (top-right):
  - Zoom-to-fit button (existing from plan 01)
  - Measurement toggle button (ruler icon, highlighted when active)
  - Screenshot button (camera icon)
  - Data table toggle button (table icon, highlighted when visible)
- Pass `onZoomToFeature` callback through to map: when triggered, compute bounds of the Nth feature in the active layer and call `map.fitBounds` on it
- The `onZoomToFeature` flow: data-table-panel calls prop -> map-visualizer passes to leaflet-map -> leaflet-map uses `useMap()` to fit bounds to the specific feature

**Update `components/leaflet-map.tsx`:**
- Accept new props: `measurementActive`, `onMeasurementToggle`, `zoomToFeatureIndex` (number | null)
- Render `MeasurementTool` component inside the map when measurement is active
- Render `ScreenshotButton` component inside the map (always, as a control)
- Add a `ZoomToFeature` sub-component that watches `zoomToFeatureIndex` prop changes and fits bounds to that feature in the active layer
