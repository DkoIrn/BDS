  <action>
    **1. API Proxy Route (`src/app/api/convert/route.ts`):**
    - Export `maxDuration = 60`
    - POST handler: auth check with `createClient()` + `getUser()`, return 401 if not authenticated
    - Forward raw request body to FastAPI (do NOT re-parse FormData -- forward raw `request.arrayBuffer()` with original `Content-Type` header to preserve multipart boundaries per Pitfall 3 in RESEARCH.md)
    - On FastAPI error: read response text, return as JSON error with matching status code
    - On success: return new NextResponse with blob body, forwarding Content-Type, Content-Disposition, X-Conversion-Warnings, X-Row-Count, X-Source-Format headers
    - Check for `FASTAPI_URL` env var, return 503 if missing

    **2. Server Page (`src/app/(dashboard)/tools/convert/page.tsx`):**
    - Simple server component that renders `<Converter />` client component
    - Keep the page title and description: "Format Converter" / "Convert between survey file formats"
    - Import and render `Converter` from `./converter`

    **3. Converter Client Component (`src/app/(dashboard)/tools/convert/converter.tsx`):**
    - `"use client"` component managing a three-step flow via state machine:
      - Step 1 (upload): Show dropzone area using react-dropzone directly (do NOT reuse FileUploadZone -- it has too many dataset-specific dependencies like createFileRecord, Supabase upload, etc. Build a simpler inline dropzone). Accept all supported extensions: .csv, .xlsx, .xls, .geojson, .json, .zip, .kml, .kmz, .xml, .dxf. Max 50MB. Show file icon + name + size after selection. Allow removing selected file to pick a different one.
      - Step 2 (select format): Show target format buttons/radio group. Smart filtering based on source file extension:
        - Geospatial inputs (.geojson, .json, .zip, .kml, .kmz, .xml, .dxf): show all 3 targets (CSV, GeoJSON, KML) minus same-format (e.g., .geojson input hides GeoJSON target, .kml input hides KML target)
        - Tabular inputs (.csv, .xlsx, .xls): show all 3 targets minus same-format for CSV (CSV input hides CSV target; Excel always shows all 3 since Excel is not an output format)
      - A "Convert" button that triggers the conversion
      - Step 3 (result): Show either success or error state
        - **Success state:** Download button (large, prominent), summary card showing: output filename, input format -> output format, row count, file size. "Convert Another" button resets to Step 1.
        - **Error state:** Inline error message with red styling, clear explanation text, "Convert Another" button.
        - **Warnings:** If X-Conversion-Warnings header is non-empty, show yellow warning banner below download button with each warning on its own line.

    - **Converting state:** Simple spinner (Loader2 from lucide-react) + status text like "Converting DXF to GeoJSON..." while fetch is in progress. No progress bar per user decision.

    - **File download trigger:** After successful fetch, create Blob from response, generate object URL, trigger download via hidden anchor element (per Pitfall 5 in RESEARCH.md). Also store the blob URL in state so the Download button can re-trigger download.

    - **State shape:**
      ```typescript
      type ConvertState =
        | { step: 'upload' }
        | { step: 'converting'; file: File; targetFormat: string }
        | { step: 'done'; file: File; targetFormat: string; blobUrl: string; outputFilename: string; rowCount: number; fileSize: number; warnings: string[] }
        | { step: 'error'; file: File; message: string }
      ```

    - **Styling:** Use Tailwind classes consistent with the existing convert page placeholder style. Card-based layout, max-w-2xl centered. Use shadcn Button component for actions. Use lucide-react icons (Upload, Download, ArrowRightLeft, FileText, AlertCircle, Loader2, Check).

    - **Cleanup:** Revoke blob URL on component unmount or when starting a new conversion (URL.revokeObjectURL).
