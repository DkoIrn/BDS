<objective>
Build the backend format conversion pipeline: three writer functions (CSV, GeoJSON, KML) that convert ParseResult to output bytes, plus a FastAPI POST /api/v1/convert endpoint that accepts file upload + target format and returns the converted file as a streaming download.

Purpose: This is the conversion engine that the frontend UI will call. It reuses existing parsers (dispatch_parser) for input and adds new writers for output, following the same pure-function pattern as the parsers.

Output: Working FastAPI endpoint at /api/v1/convert that can be tested directly with curl, plus full test coverage for all writers and the endpoint.
</objective>

  <behavior>
    CSV Writer:
    - write_csv(ParseResult with headers=["a","b"], rows=[["1","2"]]) -> CSV bytes with header row + data rows, empty warnings list
    - write_csv(ParseResult with empty rows) -> CSV bytes with header row only

    GeoJSON Writer:
    - write_geojson(ParseResult with longitude/latitude headers) -> valid GeoJSON FeatureCollection with Point features, coordinates as floats
    - write_geojson(ParseResult with no coordinate columns) -> FeatureCollection with null geometries + warning "No coordinate columns found"
    - write_geojson(ParseResult where some rows have non-numeric coordinates) -> skips bad rows, returns warning with skipped count (partial conversion per CONV-05)
    - write_geojson recognizes column name variants: longitude/lon/x/easting for X, latitude/lat/y/northing for Y, elevation/elev/z/height for Z
    - Non-coordinate columns become Feature properties

    KML Writer:
    - write_kml(ParseResult with longitude/latitude headers) -> valid KML with Document containing Placemarks with Point coordinates
    - write_kml(ParseResult with no coordinate columns) -> KML with Placemarks but no Point element + warning
    - write_kml(ParseResult with elevation) -> coordinates include altitude as "lon,lat,alt"
    - KML has proper XML declaration, kml namespace (http://www.opengis.net/kml/2.2), and ExtendedData for non-coordinate columns
    - KML adds CRS warning: "Coordinates written as-is. KML viewers expect WGS84."

    CSV/Excel parser registration:
    - dispatch_parser("data.csv", csv_bytes) -> ParseResult with headers and rows
    - dispatch_parser("data.xlsx", excel_bytes) -> ParseResult with headers and rows
  </behavior>
  <action>
    1. Create `backend/app/writers/` package with `__init__.py` exporting write_csv, write_geojson, write_kml.

    2. **csv_writer.py**: Pure function `write_csv(result: ParseResult) -> tuple[bytes, list[str]]`. Use `csv.writer` with `io.StringIO`, write headers then rows, encode to UTF-8. Return empty warnings list.

    3. **geojson_writer.py**: Pure function `write_geojson(result: ParseResult) -> tuple[bytes, list[str]]`.
       - Coordinate column detection: check headers (case-insensitive) against known sets: LON_NAMES = {"longitude", "lon", "x", "easting", "long"}, LAT_NAMES = {"latitude", "lat", "y", "northing"}, ELEV_NAMES = {"elevation", "elev", "z", "height", "altitude"}.
       - Extract `_find_column(headers, candidates) -> int | None` helper.
       - For each row: build properties dict from non-coordinate columns, build Point geometry from coordinate columns (try float conversion, skip row on ValueError and increment skipped counter).
       - If no coordinate columns found, set geometry to null and add warning.
       - If skipped > 0, add warning with count.
       - Output: `json.dumps({"type": "FeatureCollection", "features": [...]}, indent=2).encode("utf-8")`.

    4. **kml_writer.py**: Pure function `write_kml(result: ParseResult) -> tuple[bytes, list[str]]`.
       - Same coordinate column detection as geojson_writer (extract shared constants to a `backend/app/writers/_coords.py` helper to avoid duplication).
       - Use lxml.etree: create `kml` root with namespace, `Document` child, `name` element.
       - For each row: create `Placemark` with `name` (use first non-coordinate column value), `Point` > `coordinates` element as "lon,lat,alt" string.
       - Add `ExtendedData` with `Data` elements for remaining non-coordinate columns.
       - Always add warning: "Coordinates written as-is. KML viewers expect WGS84 (longitude/latitude). If your data uses projected coordinates, positions may appear incorrect."
       - Write with `xml_declaration=True, encoding="UTF-8", pretty_print=True`.

    5. **CSV/Excel parser registration**: Add CSV and Excel parsers to `PARSER_REGISTRY` in `backend/app/parsers/__init__.py`. Create two small parse functions:
       - `parse_csv_file(file_bytes: bytes) -> ParseResult`: Use Python stdlib `csv.reader` on decoded text (try utf-8-sig then latin-1). First row = headers, rest = rows.
       - `parse_excel_file(file_bytes: bytes) -> ParseResult`: Use `openpyxl.load_workbook(io.BytesIO(file_bytes), read_only=True)` to read active sheet. First row = headers (as strings), rest = rows (all values as strings).
       - Register: `.csv` -> parse_csv_file, `.xlsx` -> parse_excel_file, `.xls` -> parse_excel_file.
       - Place these in a new `backend/app/parsers/tabular_parser.py` file to keep the existing parsers clean.

    6. Write tests FIRST (RED), then implement (GREEN):
       - `test_csv_writer.py`: Test basic conversion, empty rows, special characters (commas, quotes in data).
       - `test_geojson_writer.py`: Test with coordinates, without coordinates, partial rows with bad coords, column name variants (lon/lat, x/y, easting/northing).
       - `test_kml_writer.py`: Test with coordinates (validate XML structure with lxml.etree.fromstring), without coordinates, with elevation, CRS warning presence.

    - POST /api/v1/convert with GeoJSON file + target_format=csv -> 200 with CSV bytes
    - POST /api/v1/convert with GeoJSON file + target_format=kml -> 200 with KML bytes
    - POST /api/v1/convert with unsupported target_format -> 400 with error message
    - POST /api/v1/convert with unparseable file -> 400 with error message
    - Response headers include X-Conversion-Warnings, X-Row-Count, X-Source-Format
    - Content-Disposition header has correct output filename (input basename + new extension)

  <action>
    1. Create `backend/app/routers/conversion.py`:
       - `router = APIRouter(prefix="/api/v1", tags=["conversion"])`
       - `WRITERS` dict mapping format string to tuple of (writer_fn, media_type, file_extension):
         - "csv": (write_csv, "text/csv", ".csv")
         - "geojson": (write_geojson, "application/geo+json", ".geojson")
         - "kml": (write_kml, "application/vnd.google-earth.kml+xml", ".kml")
       - `@router.post("/convert")` endpoint:
         - Params: `file: UploadFile = File(...)`, `target_format: str = Form(...)`
         - Validate target_format is in WRITERS, return 400 if not
         - Read file bytes: `file.file.read()`
         - Parse: `dispatch_parser(file_bytes, file.filename or "unknown")` -- catch ValueError and return 400
         - Write: call writer_fn(parse_result) to get (output_bytes, warnings)
         - Build output filename: strip input extension, append new extension
         - Return `StreamingResponse(io.BytesIO(output_bytes), media_type=media_type)` with headers:
           - Content-Disposition: attachment; filename="<output_name>"
           - X-Conversion-Warnings: pipe-separated warnings (or empty string)
           - X-Row-Count: str(parse_result.total_rows)
           - X-Source-Format: parse_result.source_format
       - Use sync `def` (not `async def`) for supabase-py/heavy computation compatibility per project convention.

    2. Register router in `backend/app/main.py`:
       - `from app.routers.conversion import router as conversion_router`
       - `app.include_router(conversion_router)`

       - Test GeoJSON -> KML conversion.

       - Test response headers (Content-Disposition, X-Row-Count, X-Source-Format).
       - Test partial conversion: CSV with some invalid coordinate rows -> 200 with X-Conversion-Warnings populated.
