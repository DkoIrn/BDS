  <behavior>
    - POST /api/v1/parse with {"dataset_id": "xxx"} downloads file from Supabase, dispatches to parser, stores result
    - Endpoint updates dataset with parsed_metadata, column_mappings (from headers), header_row_index=0, total_rows, status="parsed"
    - Endpoint returns 200 with {columns, preview, headerRow, totalRows, warnings}
    - Endpoint returns 400 when dataset_id is missing
    - Endpoint returns 404 when dataset not found
    - Endpoint updates status to "error" on parse failure
    - Endpoint handles unsupported file formats gracefully (status="error", descriptive message)
  </behavior>
  <action>
    Create backend/app/routers/parsing.py with a POST /api/v1/parse endpoint. Pattern follows existing validation.py router:
    - Accept {"dataset_id": str} body via Pydantic model
    - Use get_supabase_client() to fetch dataset record and download file from storage
    - Use dispatch_parser(file_bytes, dataset["file_name"]) to parse
    - Convert ParseResult to column_mappings format: each header becomes a ColumnMapping-like dict with index, originalName=header, mappedType=null (let column detection handle it later, OR run detectColumns equivalent if header names match known survey types)
    - Actually, simpler approach: store headers as first row of preview, set header_row_index=0, store column_mappings from headers. The existing column detection in the Next.js route already handles this pattern.
    - Update dataset record: status="parsed", header_row_index=0, total_rows=result.total_rows, parsed_metadata={source_format, ...result.metadata}, column_mappings from headers, parse_warnings=result.warnings or null
    - Return JSON matching existing parse route response shape: {columns: [...], preview: first 250 rows, headerRow: 0, totalRows, warnings}
    - Wrap in try/except: on failure, set status="error" and parse_warnings=[error_message] (same pattern as existing parse route)

    Register router in backend/app/main.py: import parsing router, app.include_router(parsing_router).

    Create backend/tests/test_parse_endpoint.py: Test the parse function logic (not full HTTP) by mocking supabase client. Test that dispatch + result storage works for a known format (GeoJSON). Test error handling for unsupported format. Use the existing test patterns from backend/tests/.
