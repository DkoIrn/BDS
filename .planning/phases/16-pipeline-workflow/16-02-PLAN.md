**StageImport** (`stage-import.tsx`):

Props: `StagePanelProps` (state + dispatch) plus `fileRef: React.MutableRefObject<File | null>` (to hold the actual File object in a ref, since File cannot be in reducer state).

1. Two-tab UI using local state `mode: "upload" | "existing"`:
   - Toggle buttons at top: "Upload New File" (default active) and "Select Existing Dataset".
   - Active button uses `variant="default"`, inactive uses `variant="outline"`.

2. **Upload tab:**
   - Inline react-dropzone (same pattern as converter and visualizer -- NOT using FileUploadZone which is dataset-specific).
   - Accept: `.csv, .xlsx, .xls, .geojson, .json, .zip, .kml, .kmz, .xml, .dxf` (same as converter ACCEPTED_EXTENSIONS).
   - Max 50MB.
   - On drop: store File in `fileRef.current`, dispatch `IMPORT_FILE` with `fileName: file.name`.
   - Dropzone UI: dashed border area with Upload icon, "Drag & drop a file here" text, "or click to browse" subtext. On drag active, change border color to primary.

3. **Existing dataset tab:**
   - Fetch user's recent datasets from Supabase using a server action or inline fetch to `/api/datasets` (if available) or directly via Supabase browser client.
   - Simplest approach: use `createBrowserClient()` to query `datasets` table ordered by `created_at desc` with limit 20, joined with `jobs(survey_jobs(projects(name)))` to show project context.
   - Render as a flat list of clickable rows: filename, project name as subtitle, created date, status badge.
   - On click: dispatch `IMPORT_EXISTING` with `datasetId` and `fileName`.
   - If no datasets found, show empty state: "No datasets found. Upload a file to get started."

4. If `state.stages.import.completed` (revisiting), show the summary at top: "Imported: {fileName}" with a "Re-import" button that dispatches `RESET`.

**StageInspect** (`stage-inspect.tsx`):

Props: `StagePanelProps` plus `fileRef: React.MutableRefObject<File | null>`.

1. **Auto-parse on mount** via `useEffect`:
   - If `fileSource === "upload"` and `fileRef.current` exists:
     - For CSV: use PapaParse (`Papa.parse(file, { header: false })`) to get `string[][]`.
     - For Excel: use SheetJS (`XLSX.read(buffer, { raw: false, defval: '' })`) to get `string[][]`.
     - For other formats (geojson, kml, etc.): send to `/api/parse` endpoint and receive parsed data.
     - After parsing, dispatch `INSPECT_COMPLETE` with `parsedData`, `columnCount` (first row length), and `rowCount` (data.length - 1 for header).
   - If `fileSource === "existing"` and `datasetId` exists:
     - Fetch dataset preview from Supabase: query the dataset record for `column_mappings` and fetch preview rows.
     - Dispatch `INSPECT_COMPLETE` with the fetched data.
   - Show a loading spinner (`Loader2` icon with "Parsing file..." text) while processing.

2. **After parsing (state.parsedData exists):**
   - Show summary stats in a row of cards: "Columns: N", "Rows: N", "File: {fileName}".
   - Show data preview using a simplified inline table (first 50 rows, scrollable). Use a basic `<Table>` with `TableHeader` and `TableBody` from shadcn -- do NOT import DataPreviewTable as it requires ColumnMapping which we may not have at this point. Just render raw string[][] as a plain table.
   - "Continue to Validate" primary button at bottom.
   - "Skip to Export" outline button (for users who just want format conversion).

3. If `state.stages.inspect.completed` (revisiting), show the summary and parsed data preview. "Continue" button navigates forward.

Handle errors: if parse fails, show an error alert with the message and a "Try Again" button that goes back to import.

**StageValidate** (`stage-validate.tsx`):

Props: `StagePanelProps`.

1. **Initial state** (not yet validated):
   - Show a summary of what will be validated: "{rowCount} rows, {columnCount} columns".
   - "Run QC Validation" primary button.
   - "Skip Validation" outline button with muted text underneath: "Dataset will not be checked for quality issues."

2. **On "Run QC":**
   - Dispatch `VALIDATE_START` (for loading state).
   - Call `POST /api/validate` with the dataset information. For uploaded files (no datasetId), this is tricky -- the file needs to be on the server. Simplest approach: if `fileSource === "upload"`, show a message "Upload your file to a project first to run QC validation" with a link to Projects, plus the "Skip" option. If `fileSource === "existing"`, call `/api/validate` with `{ dataset_id: datasetId }`.
   - Show loading state: Loader2 spinner with "Running quality checks..."
   - On success: dispatch `VALIDATE_COMPLETE` with `runId` and `issueCount`.

3. **After validation complete:**
   - If `issueCount === 0`: Show a success state -- ShieldCheck icon in green, "All checks passed" heading, "No quality issues detected in your dataset." subtext.
   - If `issueCount > 0`: Show issue count in a warning card -- AlertTriangle icon, "{N} issues found" heading, brief summary.
   - "Continue to Clean" primary button.
   - "Skip Clean, Go to Export" outline button.

4. **If skipped** (revisiting with `stages.validate.skipped`):
   - Show warning: AlertTriangle icon, "Validation was skipped", "Run validation now?" button.

**StageClean** (`stage-clean.tsx`):

Props: `StagePanelProps`.

1. Keep this stage simple per research pitfall #4 -- do NOT embed full transform tool flows.

2. **If validation was run and issues found:**
   - Show issue count summary from state.
   - "Remove Flagged Rows" button -- dispatches `CLEAN_COMPLETE` with a note (actual cleaning requires server-side processing; for MVP, this marks the stage as done and the export will use the original data).
   - NOTE: For true row removal, this would need a backend endpoint. For now, mark as complete and note "Flagged rows will be highlighted in export."

3. **Quick actions section** (always shown):
   - Card with links to transform tools (open in new tab):
     - "CRS Conversion" -> `/tools/transform?tool=crs` (Wrench icon)
     - "Merge Files" -> `/tools/transform?tool=merge` (Wrench icon)
     - "Split Dataset" -> `/tools/transform?tool=split` (Wrench icon)
   - Each as a link card with icon, title, and brief description.

4. **If no validation issues** (or validation skipped):
   - Show "No cleaning needed" with CheckCircle icon.
   - Transform tool links still available.

5. "Continue to Export" primary button at bottom.
6. "Skip" outline button that dispatches `SKIP_CLEAN`.

**StageExport** (`stage-export.tsx`):

Props: `StagePanelProps` plus `fileRef: React.MutableRefObject<File | null>`.

1. **Format selection:**
   - Radio-button style cards for output formats: CSV, GeoJSON, KML (same as converter TARGET_FORMATS).
   - Default selection: match input file extension, fallback to CSV.
   - On select, dispatch `SET_EXPORT_FORMAT`.

2. **Download section:**
   - "Download Dataset" primary button.
   - On click: If `fileSource === "upload"` and `fileRef.current` exists, send to `/api/convert` with the file and selected format (same pattern as converter.tsx fetch). If `fileSource === "existing"`, send dataset_id to convert endpoint.
   - Show download progress with Loader2 spinner.
   - On success: trigger download via blob URL + hidden anchor pattern (same as converter).
   - Show success state: Check icon, "Downloaded: {filename}" with file size.

3. **QC Report download** (optional):
   - If validation was run, show "Download QC Report (PDF)" secondary button.
   - Calls `/api/reports/pdf?dataset_id={id}` (existing endpoint).
   - Only shown when `state.validationRunId` exists.

4. **Pipeline complete state:**
   - After download, show a completion summary card with all stage summaries.
   - "Start New Pipeline" button that dispatches `RESET` and clears sessionStorage.
