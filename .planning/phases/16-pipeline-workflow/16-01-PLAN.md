Create `pipeline-state.ts` with:

1. **Types:**
   - `PipelineStage` union: `"import" | "inspect" | "validate" | "clean" | "export"`
   - `StageStatus` interface: `{ completed: boolean; skipped: boolean; summary: string | null }`
   - `PipelineState` interface with flat structure:
     - `currentStage: PipelineStage`
     - `stages: Record<PipelineStage, StageStatus>`
     - `fileName: string | null` (stored for display -- NOT the File object)
     - `fileSource: "upload" | "existing" | null`
     - `datasetId: string | null`
     - `parsedData: string[][] | null`
     - `columnCount: number | null`
     - `rowCount: number | null`
     - `validationRunId: string | null`
     - `issueCount: number | null`
     - `cleanedData: Blob | null` (not serialized to sessionStorage)
     - `exportFormat: string | null`
   - `PipelineAction` discriminated union:
     - `IMPORT_FILE` with `{ fileName: string }` (name only, File kept in component ref)
     - `IMPORT_EXISTING` with `{ datasetId: string; fileName: string }`
     - `INSPECT_COMPLETE` with `{ parsedData: string[][]; columnCount: number; rowCount: number }`
     - `SKIP_VALIDATE`
     - `VALIDATE_START`
     - `VALIDATE_COMPLETE` with `{ runId: string; issueCount: number }`
     - `SKIP_CLEAN`
     - `CLEAN_COMPLETE` with `{ cleanedData: Blob }`
     - `SET_EXPORT_FORMAT` with `{ format: string }`
     - `GO_TO_STAGE` with `{ stage: PipelineStage }`
     - `RESET`

2. **initialState** constant with all stages as `{ completed: false, skipped: false, summary: null }` and currentStage: "import".

3. **pipelineReducer(state, action)** -- pure function with exhaustive switch:
   - `IMPORT_FILE`: Reset to initial, set currentStage to "inspect", mark import as completed with fileName as summary, set fileSource to "upload", set fileName.
   - `IMPORT_EXISTING`: Same but fileSource "existing", set datasetId.
   - `INSPECT_COMPLETE`: Set parsedData, columnCount, rowCount, mark inspect completed with summary like "N columns, M rows", set currentStage to "validate".
   - `SKIP_VALIDATE`: Mark validate as skipped with summary "Skipped -- dataset not validated", set currentStage to "clean".
   - `VALIDATE_START`: Just stay on validate (for loading state tracking in the component).
   - `VALIDATE_COMPLETE`: Set validationRunId, issueCount, mark validate completed with summary like "N issues found" or "All checks passed", set currentStage to "clean".
   - `SKIP_CLEAN`: Mark clean as skipped, set currentStage to "export".
   - `CLEAN_COMPLETE`: Set cleanedData, mark clean completed, set currentStage to "export".
   - `SET_EXPORT_FORMAT`: Set exportFormat.
   - `GO_TO_STAGE`: Only transition if `canNavigateTo` returns true.
   - `RESET`: Return initialState.

4. **canNavigateTo(state, targetStage)** function implementing smart gating:
   - `import`: Always navigable (restart).
   - `inspect`: Only if import is completed.
   - `validate`: Only if inspect is completed.
   - `clean`: Only if inspect is completed (validate can be skipped).
   - `export`: Only if import is completed (users can export even without validation per CONTEXT.md).

5. **STAGE_ORDER** constant array for index-based comparison.

Create `pipeline-store.ts` with (following session-store.ts pattern):

1. `PIPELINE_KEY = "pipeline-workflow-state"` constant.

2. `SerializablePipelineState` type that omits `parsedData` and `cleanedData` (non-serializable/large).

3. `savePipelineState(state: PipelineState): void` -- serialize only the serializable fields to sessionStorage. Wrap in try/catch.

4. `loadPipelineState(): Partial<PipelineState> | null` -- parse from sessionStorage, return null if missing or invalid. Wrap in try/catch.

5. `clearPipelineState(): void` -- remove from sessionStorage.

Export all types and functions.
