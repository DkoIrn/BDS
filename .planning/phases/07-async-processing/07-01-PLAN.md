   - Import `BackgroundTasks` from `fastapi`
   - Change `@router.post("/validate", response_model=ValidateResponse)` to `@router.post("/validate", status_code=202)`
   - Add `background_tasks: BackgroundTasks` parameter to `validate_dataset()`
   - In the endpoint body: only verify dataset exists (quick SELECT by id), then call `background_tasks.add_task(run_validation_background, request.dataset_id, request.config)`
   - Return `{"status": "accepted", "dataset_id": request.dataset_id}`
   - Do NOT set status to 'validating' in FastAPI -- the Next.js route handles that (avoids race condition per research pitfall 5)

   - Move the entire try/except block from the current endpoint into this function
   - The function gets its own Supabase client via `get_supabase_client()`
   - Remove the `supabase.table("datasets").update({"status": "validating"})` line (Next.js sets this)
   - Keep the `supabase.table("datasets").update({"status": "validated"})` on success
   - Keep the `supabase.table("datasets").update({"status": "validation_error"})` on failure
   - Wrap the ENTIRE function body in try/except Exception to ensure status always gets updated (pitfall 4)
   - Do NOT re-raise the exception -- BackgroundTasks exceptions are logged by Starlette but should not crash anything
   - The function does NOT return anything (it writes results to Supabase directly)

3. Update the response model: create a simple `AcceptedResponse` model or return a plain dict `{"status": "accepted", "dataset_id": request.dataset_id}`.
