  <files>
    src/lib/actions/auth.ts,

  <action>
    1. Create `src/lib/actions/auth.ts` with 'use server' directive containing these server actions:

       **signup(formData: FormData)**: Extract email and password. Call `supabase.auth.signUp()` with `emailRedirectTo` set to `${process.env.NEXT_PUBLIC_SITE_URL}/auth/callback`. On error, return `{ error: error.message }`. On success, redirect to `/login?message=Check your email to verify your account`.

       **login(formData: FormData)**: Extract email and password. Call `supabase.auth.signInWithPassword()`. On error, return `{ error: error.message }`. On success, redirect to `/dashboard`.

       **logout()**: Call `supabase.auth.signOut()`. Redirect to `/login`.

       **resetPassword(formData: FormData)**: Extract email. Call `supabase.auth.resetPasswordForEmail(email, { redirectTo: '${process.env.NEXT_PUBLIC_SITE_URL}/auth/callback?next=/update-password' })`. On error, return `{ error: error.message }`. On success, redirect to `/login?message=Check your email for a password reset link`.

       **updatePassword(formData: FormData)**: Extract password and confirmPassword. Validate they match (return error if not). Call `supabase.auth.updateUser({ password })`. On error, return `{ error: error.message }`. On success, redirect to `/dashboard`.

       All actions use the server Supabase client from `@/lib/supabase/server`.

    2. Create `src/app/auth/callback/route.ts` following Pattern 4 from research:
       - Export GET handler
       - Extract `token_hash` and `type` from search params
       - Call `supabase.auth.verifyOtp({ type, token_hash })`
       - On success, redirect to `next` param (default `/dashboard`)
       - On failure, redirect to `/auth/error`

    3. Create `src/app/auth/error/page.tsx`:
       - Simple page displaying "Something went wrong" with a link back to /login
       - Use Card component from shadcn/ui for the error message container
       - Style with brand colors
  </action>
