  <action>
    1. Create `src/lib/supabase/client.ts` -- Browser client using `createBrowserClient` from `@supabase/ssr`:
       ```typescript
       import { createBrowserClient } from '@supabase/ssr'

       export function createClient() {
         return createBrowserClient(
           process.env.NEXT_PUBLIC_SUPABASE_URL!,
           process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
         )
       }
       ```

    2. Create `src/lib/supabase/server.ts` -- Server client using `createServerClient` from `@supabase/ssr` with cookie handling. Follow the exact pattern from research (Pattern 2): uses `cookies()` from `next/headers`, implements `getAll()` and `setAll()` with try/catch for Server Component compatibility.

    3. Create `src/middleware.ts` -- Auth middleware following Pattern 3 from research:
       - Create Supabase server client with cookie forwarding
       - Call `getUser()` (NOT `getSession()` -- security requirement)
       - Redirect unauthenticated users to `/login` (except for auth routes: `/login`, `/signup`, `/forgot-password`, `/update-password`, `/auth`)
       - Redirect authenticated users away from `/login` and `/signup` to `/dashboard`
       - Matcher excludes static assets: `/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)`

    4. Create `src/types/database.ts` -- TypeScript types for the profiles table:
       ```typescript
       export interface Profile {
         id: string
         email: string | null
         full_name: string | null
         created_at: string
         updated_at: string
       }
       ```

    5. Create `supabase/migrations/00001_profiles.sql` with the full migration from research:
       - `public.profiles` table (id UUID referencing auth.users, email, full_name, created_at, updated_at)
       - RLS enabled with policies for own-profile SELECT and UPDATE
       - `handle_new_user()` trigger function that auto-creates a profile on auth.users INSERT
       - `handle_updated_at()` trigger function for automatic updated_at timestamps
       This migration is applied manually via Supabase Dashboard SQL editor (documented in user_setup).
  </action>
