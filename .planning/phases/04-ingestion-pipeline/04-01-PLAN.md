    - detectHeaderRow on rows with single-cell metadata rows skips them
    - detectHeaderRow defaults to 0 if no clear header found

    Column Detector:
    - detectColumns with headers ["KP", "Easting", "Northing"] returns types kp, easting, northing with high confidence
    - detectColumns with headers ["chainage", "E", "N"] matches kp, easting, northing via regex patterns
    - detectColumns with unrecognizable header "Foo" returns null type with low confidence
    - detectColumns with numeric-only column data and ambiguous header uses data sampling to detect type
    - detectColumns with surveyType 'DOB' and missing 'dob' column is reported by getMissingExpectedColumns
    - Confidence is 'high' when both name and data match, 'medium' when only name matches, 'low' when only data matches or neither
  </behavior>
  <action>
    1. Create src/lib/parsing/header-detector.ts:
       - Export detectHeaderRow(rows: string[][]): number
       - Scan first 20 rows using heuristics:
         a. Skip rows where most cells are empty (>70% empty)
         b. Skip rows that look like metadata: single non-empty cell, or "key: value" pattern
         c. Find first row where: most cells have text content (not pure numbers), cell count is consistent with subsequent rows, and the row has at least 3 non-empty cells
       - Return 0 as default if no clear header found

    2. Create src/lib/parsing/column-detector.ts:

       - Export detectColumns(rows: string[][], headerRowIndex: number, surveyType: SurveyType): DetectedColumn[]
         a. Extract headers from rows[headerRowIndex]
         b. Extract data sample: rows[headerRowIndex+1] through [headerRowIndex+100]
         c. For each header, run matchByName (regex patterns) and matchByData (data sampling)
         d. matchByData: check if column values are all numeric, all monotonically increasing (kp candidate), coordinate-range (easting/northing), small positive numbers (depth/dob/doc)
         e. Combine name + data evidence for confidence: both match = 'high', name only = 'medium', data only = 'low', neither = null type
       - Export getMissingExpectedColumns(detected: DetectedColumn[], surveyType: SurveyType): SurveyColumnType[]
         a. Define EXPECTED_COLUMNS per survey type (DOB needs kp+dob+easting+northing, etc.)
