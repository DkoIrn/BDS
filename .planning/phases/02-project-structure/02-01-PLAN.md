2. Create `supabase/migrations/00002_projects_jobs.sql` with:
   - `projects` table: id (UUID PK, gen_random_uuid), user_id (FK auth.users ON DELETE CASCADE, NOT NULL), name (TEXT NOT NULL), description (TEXT nullable), status (TEXT NOT NULL DEFAULT 'active'), created_at (TIMESTAMPTZ DEFAULT NOW), updated_at (TIMESTAMPTZ DEFAULT NOW)
   - `jobs` table: id (UUID PK, gen_random_uuid), project_id (FK projects ON DELETE CASCADE, NOT NULL), user_id (FK auth.users ON DELETE CASCADE, NOT NULL), name (TEXT NOT NULL), survey_type (TEXT NOT NULL), description (TEXT nullable), status (TEXT NOT NULL DEFAULT 'pending'), created_at (TIMESTAMPTZ DEFAULT NOW), updated_at (TIMESTAMPTZ DEFAULT NOW)
   - Indexes on projects(user_id), jobs(project_id), jobs(user_id)
   - RLS enabled on both tables with SELECT/INSERT/UPDATE/DELETE policies using auth.uid() = user_id
   - Reuse the existing `handle_updated_at()` trigger function (do NOT recreate it) — just create triggers on both tables
   - Status values — projects: active/completed/archived, jobs: pending/processing/completed/failed

3. Create `src/lib/types/projects.ts` with:
   - SURVEY_TYPES const array: ['DOB', 'DOC', 'TOP', 'Event Listing', 'Pipeline Position', 'ROVV'] as const
   - SurveyType type derived from SURVEY_TYPES
   - PROJECT_STATUSES const: ['active', 'completed', 'archived'] as const
   - ProjectStatus type
   - JOB_STATUSES const: ['pending', 'processing', 'completed', 'failed'] as const
   - JobStatus type
   - Project interface: id, user_id, name, description (string|null), status (ProjectStatus), created_at, updated_at, jobs ({count: number}[])
   - Job interface: id, project_id, user_id, name, survey_type (SurveyType), description (string|null), status (JobStatus), created_at, updated_at
  </action>
